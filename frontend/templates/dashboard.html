Formal Representation of Finite Automata (5-Tuple)

Both NFA and DFA are formally defined using a 5-tuple, which provides a complete mathematical description of the automaton.

General Form

M = (Q, Σ, δ, q₀, F)

  • Q: Finite set of states
  • Σ (Sigma): Input alphabet (set of symbols)
  • δ (delta): Transition function
  • q₀: Start state
  • F: Set of final (accepting) states

Non-Deterministic Finite Automaton (NFA)

An NFA allows multiple transitions for the same input symbol from a state. It may also include epsilon (ε) transitions.

  • One input → multiple possible states
  • Can include ε-transitions
  • Easy to design but not directly implementable

Formal Definition

(Q, Σ, δ, q₀, F) where:

δ : Q × Σ → 2Q

  • For a given state and input, the result is a set of possible states
  • Multiple transitions are allowed
  • May include ε-transitions (optional)

Deterministic Finite Automaton (DFA)

A DFA has exactly one transition for each input symbol from every state.

  • No ambiguity in transitions
  • No epsilon transitions
  • Fully deterministic

Formal Definition

δ : Q × Σ → Q

  • For every state and input, there is exactly one next state
  • No ambiguity in transitions
  • No ε-transitions

NFA to DFA Conversion (Subset Construction)

Subset construction converts an NFA into an equivalent DFA by treating sets of NFA states as individual DFA states.

Steps:

  1. Start with initial state {q₀}
  2. Compute reachable states for each input
  3. Create new DFA states as sets
  4. Repeat until no new states appear

Key Idea: Each DFA state represents a subset of NFA states.
